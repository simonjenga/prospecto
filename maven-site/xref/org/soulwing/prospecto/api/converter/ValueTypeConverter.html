19  package org.soulwing.prospecto.api.converter;
20  
21  /**
22   * A converter that transforms simple model objects into a suitable
23   * representation for a view node of value type and vice-versa.
24   * <p>
25   * View nodes of value type have a limited range of supported data types
26   * (String, Number, Boolean, Enum). A converter allows other model types to be
27   * represented in value nodes.
28   * <p>
29   * Almost any model type that has a well-defined lexical representation can be
30   * transformed using a converter so that it may be used in view node of value
31   * type. For example, your model might define a Money type that describes an
32   * amount of money in a given currency. You could make a converter that
33   * creates a string representation of the Money type; the string consists of
34   * both the number representing the amount and the currency designator.
35   * <p>
36   * A converter's {@link #supports(Class)} method is used to determine whether
37   * a given model type can be converted. This method could be implemented using
38   * {@link #equals(Object)} in which case the converter supports exactly one
39   * model type (in particular such a converter rejects subclasses of the model
40   * type). Alternatively, the {@link #supports(Class)} method could be
41   * implemented using {@link Class#isAssignableFrom(Class)}, in which case the
42   * converter claims support for the given type and <em>all</em> of its subtypes.
43   * There may be other useful strategies in determining whether to support a
44   * given model type; for example, all model types with a particular annotation.
45   * In any case, <em>care must be taken in choosing both the base type to
46   * support and the order in which converters are consulted</em>, so that the
47   * appropriate converter is used for given model type.
48   * <p>
49   * A converter <em>must</em> have a no-arg constructor in order to fully support
50   * all view template configurations. The framework provides the ability to
51   * configure the converter after it is constructed by injecting values of
52   * simple types via JavaBeans-style accessors. Additionally the framework will
53   * invoke any methods annotated with {@link javax.annotation.PostConstruct}
54   * after any configuration properties have been injected.
55   * <p>
56   * A converter instance will be used to perform an arbitrary number of type
57   * conversions by multiple concurrent threads. The best way to satisfy these
58   * requirements is for the converter to use a <em>stateless</em> design; i.e.
59   * it should not modify <em>any</em> instance variable after the converter is
60   * created and configured. Designs using thread synchronization mechanisms are
61   * also possible, but may have a significant impact on view generation
62   * performance.
63   *
64   * @param <V> value type; must be one of String, Number (or any of its
65   * subtypes), Boolean, Enum, or Date (or any of its subtypes)
66   *
67   * @author Carl Harris
68   */
69  public interface ValueTypeConverter<V> {
70  
71    /**
72     * Tests whether this converter supports a given model type.
73     * @param type model type
74     * @return {@code true} if this converter can transform between an instance of
75     *    {@code type} and an instance of type {@code V}
76     */
77    boolean supports(Class<?> type);
78  
79    /**
80     * Gets the view type produced by this converter
81     * @return view type; which must be one of String, Boolean, Number, or Date
82     */
83    Class<?> getViewType();
84  
85    /**
86     * Converts a model object (generally of a simple type) to a suitable
87     * representation for a view node of value type.
88     * @param model the subject model object (never {@code null})
89     * @return value node representation of {@code model}
90     * @throws Exception if conversion fails
91     */
92    V toValue(Object model) throws Exception;
93  
94    /**
95     * Converts a value from a view into a model object.
96     * @param value value node representation (never {@code null})
97     * @return model object
98     * @throws Exception if conversion fails
99     */
100   Object toObject(Object value) throws Exception;
101 
102 }
